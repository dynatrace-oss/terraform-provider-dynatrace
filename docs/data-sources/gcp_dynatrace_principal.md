# dynatrace_gcp_dynatrace_principal (Data Source)

The `dynatrace_gcp_dynatrace_principal` data source returns the GCP service account email that Dynatrace uses to impersonate the customer service account configured on a [`dynatrace_gcp_connection`](../resources/gcp_connection.md). The principal is **per tenant**, fixed for the lifetime of the tenant, and is the same value that `dtctl get gcp principal` prints.

The data source is read-only with respect to user configuration; on first call against a fresh tenant it will trigger the platform to materialize the principal (an empty `builtin:hyperscaler-authentication.connections.gcp-dynatrace-principal` Settings 2.0 object is created on demand and the resulting `principal` is read back). Subsequent calls just read the existing entry.

Use the returned `principal` value as the `member` of a `google_service_account_iam_member` binding granting `roles/iam.serviceAccountTokenCreator` on the customer service account.

## Example Usage

```terraform
data "dynatrace_gcp_dynatrace_principal" "this" {}

resource "google_service_account_iam_member" "dynatrace_token_creator" {
  service_account_id = google_service_account.monitoring.name
  role               = "roles/iam.serviceAccountTokenCreator"
  member             = "serviceAccount:${data.dynatrace_gcp_dynatrace_principal.this.principal}"
}
```
